# CmakeCpp

## 概要
cmakeを用いたC++のプロジェクト。

## 用途
定期的にひっかかるデストラクタの仕様。特に実行順序について。
非同期処理をすると他のクラスのポインタを渡したり、コールバック関数を登録したりすることが多い。
その際に、デストラクタの実行順序を誤ると、解放済みのポインタにアクセスしてしまい、未定義動作を引き起こすことがある。
デストラクタの実行順序の確認と、安全な設計方法を模索するためのプロジェクト。

### TestOrder
TestOrderクラスは、デストラクタの実行順序を確認するためのクラス。

### TestAsync
非同期処理をデストラクタ任せに止めると危険な場合が有ることを示すクラス。
